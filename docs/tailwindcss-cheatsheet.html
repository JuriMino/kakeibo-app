1. 基本の考え方

Tailwind は 「クラス名 = CSSプロパティ1つ」のユーティリティCSS。 クラスを組み合わせるだけでスタイルが完成する。 

<!-- 例: 白背景 + 角丸 + 影 + パディング -->
<div class="bg-white rounded-lg shadow p-6">
    支出カード
</div>
命名のパターン:
  • {プロパティ}-{値} → p-4(padding), text-sm(文字サイズ)
  • {プロパティ}-{色}-{濃さ} → bg-emerald-600, text-gray-700
  • {画面幅}:{クラス} → md:flex(中サイズ以上で flex)
  • {状態}:{クラス} → hover:bg-blue-700(ホバー時)

2. レイアウト（Flexbox / Grid）

Flexbox

クラス意味
flex要素を横並びに（display: flex）
inline-flexインラインの flex
flex-col縦方向に並べる
items-center交差軸（縦）中央揃え
items-start / items-end上揃え / 下揃え
justify-center主軸（横）中央揃え
justify-between両端寄せ（間にスペース）
justify-end右寄せ
space-x-2 / space-y-4子要素の間隔（横/縦）
<!-- 「合計」と「新規追加ボタン」を両端に配置（家計簿で使用） -->
<div class="flex items-center justify-between">
    <p>合計支出額</p>
    <a href="...">+ 新規追加</a>
</div>

Grid

クラス意味
griddisplay: grid
grid-cols-22列
grid-cols-33列
gap-4セル間の余白
<!-- 金額・支出日を2列に並べる -->
<div class="grid grid-cols-2 gap-4">
    <div>金額</div>
    <div>支出日</div>
</div>
⚠️ grif-cols-2 のような typo に注意（kakeibo-appの_form.blade.phpに存在）。 Tailwindはクラス名が間違っていても無視するだけでエラーは出ないので、効かない時は綴りを疑う。

3. 余白（padding / margin / gap）

クラス意味
p-4上下左右に padding
px-4左右の padding
py-2上下の padding
pt-4 / pb-4 / pl-4 / pr-4上 / 下 / 左 / 右 のみ
m-4 / mx-auto / mt-1margin（mx-autoは左右中央寄せ）
gap-4flex / grid の子要素間
space-x-2 / space-y-4子要素間の余白（横/縦）
数値の目安: 1 = 4px, 2 = 8px, 4 = 16px, 6 = 24px, 8 = 32px。 「p-4って何px？」と迷ったら4倍を思い出す。

4. サイズ（width / height）

クラス意味
w-full幅100%
w-1/2幅50%
w-64幅 64 × 4 = 256px
max-w-2xl最大幅 42rem (約672px)
max-w-5xl最大幅 64rem (約1024px)
min-w-full最小幅100%（テーブルでよく使う）
h-screen高さ画面いっぱい

5. テキスト

クラス意味
text-xs0.75rem (12px)
text-sm0.875rem (14px)
text-base1rem (16px) ※デフォルト
text-lg / text-xl / text-3xl大きめの文字

パターン

7. 枠線・角丸・影

クラス意味
border1pxの枠線
border-22px
border-gray-300枠線色
divide-y子要素の間に横線（テーブル行区切りで使う）
rounded少し角丸 (4px)
rounded-md中くらい (6px)
rounded-lg大きめ (8px)
rounded-full完全な丸（バッジで使用）

8. ホバー・フォーカス・状態

クラスの前にプレフィックスをつけることで、状態に応じたスタイルが当たる。

プレフィックス意味
hover:マウスオーバー時
focus:クリック・フォーカス時
active:押下中
disabled:disabled属性が付いている時
<a class="text-blue-600 hover:underline">編集</a>
<button class="bg-emerald-600 hover:bg-emerald-700">登録</button>

フォーカスリング（入力欄でよく使う）

<input class="focus:outline-none focus:ring-2 focus:ring-offset-2 focus:ring-emerald-500">
クラス意味
focus:outline-noneブラウザのデフォルト枠線を消す
focus:ring-22pxのリングを表示
focus:ring-offset-2リングを少し離す
focus:ring-emerald-500リングの色
focus:border-emerald-500枠線色も変える

9. フォーム部品

kakeibo-appの _form.blade.php でも使っている定番セット。

<input type="text"
       class="mt-1 w-full rounded-md border-gray-300 shadow-sm
              focus:border-emerald-500 focus:ring-emerald-500">
クラス意味
mt-1ラベルとの間に少し余白
w-full幅100%
rounded-md角丸
border-gray-300淡い枠線
shadow-sm軽い影
focus:border-emerald-500フォーカス時に枠線を緑に
focus:ring-emerald-500フォーカス時に緑のリング

ボタン

<button class="inline-flex items-center px-4 py-2
               bg-emerald-600 hover:bg-emerald-700
               text-white text-sm font-medium
               rounded-md shadow">
    登録する
</button>

10. テーブル

<table class="min-w-full divide-y divide-gray-200">
    <thead class="bg-gray-50">
        <tr>
            <th class="px-4 py-3 text-left text-xs font-semibold text-gray-600 uppercase">日付</th>
        </tr>
    </thead>
    <tbody class="divide-y divide-gray-100">
        <tr>
            <td class="px-4 py-3 text-sm text-gray-700">2026/05/06</td>
        </tr>
    </tbody>
</table>
クラス意味
min-w-fullテーブルを親幅いっぱいに
divide-y各行の間に横線
divide-gray-200その線の色
⚠️ th と td の text-* は揃える。kakeibo-appでヘッダー「操作」を text-left、ボディの「編集/削除」を text-center にしていて見た目がズレた事例あり。

11. レスポンシブ

画面幅でスタイルを切り替えるには、クラス名にプレフィックスを付ける。

プレフィックス適用される画面幅用途の目安
sm:≧ 640px小型タブレット以上
md:≧ 768pxタブレット以上
lg:≧ 1024pxノートPC以上
xl:≧ 1280pxデスクトップ
<!-- スマホは padding を狭く、タブレット以上で広く -->
<div class="px-2 sm:px-6 lg:px-8">...</div>
モバイルファースト: プレフィックスなしのクラスは「すべての画面サイズ」に当たる。 sm:などは「その幅以上」で上書きされる。スマホ用 → 画面が大きくなったら追加 で書く。
